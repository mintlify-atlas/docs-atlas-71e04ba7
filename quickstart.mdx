## Prerequisites

- Docker and Docker Compose installed on your system
- A media library (local files or existing Plex/Jellyfin/Emby server)
- At least 128 MB free disk space for configuration
- Additional storage for transcoding (tmpfs recommended for performance)

## Quick Start with Docker

The fastest way to get ErsatzTV running is with Docker Compose.

<Steps>
  <Step title="Create docker-compose.yml">
    Create a `docker-compose.yml` file with the following content:

    ```yaml docker-compose.yml
    services:
      ersatztv:
        image: ersatztv/ersatztv:latest
        container_name: ersatztv
        environment:
          - TZ=America/Chicago  # Set your timezone
        ports:
          - "8409:8409"  # Web UI and streaming
        volumes:
          - ersatztv_config:/config
          - /path/to/your/media:/media:ro  # Mount your media (read-only)
        tmpfs:
          - /transcode  # Fast temporary storage for transcoding
        restart: unless-stopped

    volumes:
      ersatztv_config:
    ```

    <Note>
      Replace `/path/to/your/media` with the actual path to your media library. Remove this volume if you're using Plex/Jellyfin/Emby.
    </Note>
  </Step>

  <Step title="Start the container">
    Run ErsatzTV:

    ```bash
    docker-compose up -d
    ```

    Watch the logs to ensure it starts successfully:

    ```bash
    docker-compose logs -f ersatztv
    ```

    <Tip>
      First startup will take a moment as ErsatzTV initializes the database and configuration folders.
    </Tip>
  </Step>

  <Step title="Access the Web UI">
    Open your browser and navigate to:

    ```
    http://localhost:8409
    ```

    You should see the ErsatzTV web interface.
  </Step>

  <Step title="Add a Media Source">
    Choose one of the following options:

    <Tabs>
      <Tab title="Local Media">
        1. Navigate to **Media Sources** → **Local Libraries**
        2. Click **Add Library**
        3. Select the library type (Movies, Shows, Music Videos, etc.)
        4. Add the path where you mounted your media (e.g., `/media/movies`)
        5. Click **Scan Library** to index your content
      </Tab>

      <Tab title="Plex">
        1. Navigate to **Media Sources** → **Plex**
        2. Click **Add Plex Media Source**
        3. Enter your Plex server details
        4. Select libraries to synchronize
        5. Wait for the initial sync to complete
      </Tab>

      <Tab title="Jellyfin">
        1. Navigate to **Media Sources** → **Jellyfin**
        2. Click **Add Jellyfin Media Source**
        3. Enter your Jellyfin server URL and API key
        4. Select libraries to synchronize
        5. Wait for the initial sync to complete
      </Tab>

      <Tab title="Emby">
        1. Navigate to **Media Sources** → **Emby**
        2. Click **Add Emby Media Source**
        3. Enter your Emby server details
        4. Select libraries to synchronize
        5. Wait for the initial sync to complete
      </Tab>
    </Tabs>
  </Step>

  <Step title="Create Your First Channel">
    Now let's create a simple channel:

    1. Navigate to **Channels** in the left menu
    2. Click **Add Channel**
    3. Configure basic settings:
       - **Number**: 1 (or any channel number you prefer)
       - **Name**: My First Channel
       - **Categories**: General (optional)
    4. Click **Save**
  </Step>

  <Step title="Create a Schedule">
    Channels need schedules to know what content to play:

    1. Navigate to **Schedules** → **Classic Schedules**
    2. Click **Add Schedule**
    3. Give it a name like "All Day Movies"
    4. Click **Add Schedule Item**
    5. Configure the item:
       - **Start Type**: Fixed (Start at a specific time)
       - **Start Time**: 00:00 (midnight)
       - **Collection**: Choose a collection or library
       - **Playback Order**: Shuffle, Sequential, or other
    6. Click **Save**
  </Step>

  <Step title="Create a Playout">
    Connect your channel to the schedule:

    1. Navigate to **Playouts**
    2. Click **Add Playout**
    3. Select:
       - **Channel**: My First Channel
       - **Schedule**: All Day Movies
    4. Click **Save**
    5. Click **Build Playout** to generate the schedule
  </Step>

  <Step title="Start Watching">
    Your channel is now live! Access it via:

    **Direct Stream URL:**
    ```
    http://localhost:8409/iptv/channel/1.m3u8
    ```

    **M3U Playlist** (for all channels):
    ```
    http://localhost:8409/iptv/playlist.m3u8
    ```

    **XMLTV EPG:**
    ```
    http://localhost:8409/iptv/xmltv.xml
    ```

    <Tip>
      Use these URLs in any IPTV player like VLC, Kodi, or dedicated IPTV apps.
    </Tip>
  </Step>
</Steps>

## Test Your Channel

### Using VLC

1. Open VLC Media Player
2. Go to **Media** → **Open Network Stream**
3. Paste your channel URL: `http://localhost:8409/iptv/channel/1.m3u8`
4. Click **Play**

### Using the Built-in Preview

1. In the ErsatzTV web UI, go to **Channels**
2. Find your channel in the list
3. Click the preview button (if your browser supports HEVC/H.264)

## Troubleshooting

<AccordionGroup>
  <Accordion title="Can't access the web UI">
    - Verify the container is running: `docker-compose ps`
    - Check logs for errors: `docker-compose logs ersatztv`
    - Ensure port 8409 is not already in use
    - Try accessing via server IP instead of localhost
  </Accordion>

  <Accordion title="Channel won't play">
    - Verify the playout has been built (check **Playouts** page)
    - Check that your schedule has content scheduled
    - Review logs: `docker-compose logs -f ersatztv`
    - Use the **Troubleshoot Playback** feature in the UI
  </Accordion>

  <Accordion title="Media not appearing after scan">
    - Check that media paths are correctly mounted in Docker
    - Verify file permissions (container needs read access)
    - Review scanner logs: `docker-compose logs -f ersatztv`
    - Try a manual deep scan from the UI
  </Accordion>

  <Accordion title="Playback is stuttering or buffering">
    - Check CPU/GPU usage during transcoding
    - Consider enabling hardware acceleration (see Installation)
    - Adjust FFmpeg profile quality settings
    - Ensure tmpfs is configured for `/transcode`
  </Accordion>
</AccordionGroup>
