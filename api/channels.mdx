## Overview

Channels are the core of ErsatzTV, representing virtual TV channels with scheduled content. The Channels API allows you to retrieve channel information and control channel playouts.

## Get All Channels

Retrieve a list of all configured channels.

```bash
GET /api/channels
```

### Response

<ResponseField name="channels" type="array">
  Array of channel objects
</ResponseField>

<Expandable title="Channel Object Properties">
  <ResponseField name="id" type="integer" required>
    Unique channel identifier
  </ResponseField>
  
  <ResponseField name="number" type="string" required>
    Channel number (e.g., "1", "2.1", "100")
  </ResponseField>
  
  <ResponseField name="name" type="string" required>
    Channel name displayed in guides
  </ResponseField>
  
  <ResponseField name="ffmpegProfile" type="string" required>
    Name of the FFmpeg profile used for transcoding
  </ResponseField>
  
  <ResponseField name="language" type="string" required>
    Primary language code (e.g., "eng", "spa")
  </ResponseField>
  
  <ResponseField name="streamingMode" type="string" required>
    Streaming mode for the channel. Values:
    - `HttpLiveStreamingDirect` - HLS direct streaming
    - `HttpLiveStreamingSegmenter` - HLS with segmenter
    - `TransportStream` - MPEG-TS legacy mode
    - `TransportStreamHybrid` - MPEG-TS hybrid mode
  </ResponseField>
</Expandable>

## Reset Channel Playout

Reset a channel's playout schedule, triggering a rebuild from the current time.

```bash
POST /api/channels/{channelNumber}/playout/reset
```

### Path Parameters

<ParamField path="channelNumber" type="string" required>
  The channel number to reset (e.g., "1", "2.1")
</ParamField>

### Response

<ResponseField name="status" type="integer">
  - `200 OK` - Playout reset queued successfully
  - `404 Not Found` - Channel does not exist
</ResponseField>

### Example Request

```bash
curl -X POST http://localhost:8409/api/channels/1/playout/reset
```

### When to Reset Playouts

Resetting a playout is useful when:

- Schedule changes need to be applied immediately
- Content in collections has been updated
- Playout appears stuck or out of sync
- Testing schedule configurations

<Note>
  Resetting a playout rebuilds the schedule from the current time forward. Active streams may be interrupted briefly during the reset.
</Note>

## Implementation Details

From `ErsatzTV/Controllers/Api/ChannelController.cs:24-34`:

```csharp
public async Task<IActionResult> ResetPlayout(string channelNumber)
{
    Option<int> maybePlayoutId = await mediator.Send(
        new GetPlayoutIdByChannelNumber(channelNumber)
    );
    
    foreach (int playoutId in maybePlayoutId)
    {
        await workerChannel.WriteAsync(
            new BuildPlayout(playoutId, PlayoutBuildMode.Reset)
        );
        return new OkResult();
    }
    
    return new NotFoundResult();
}
```

The reset operation:

1. Looks up the playout ID associated with the channel number
2. Queues a `BuildPlayout` background task with `PlayoutBuildMode.Reset`
3. Returns immediately while the rebuild happens asynchronously

## Streaming Modes

Channels support different streaming modes optimized for various clients:
