## Overview

ErsatzTV emulates HDHomeRun network tuner devices, allowing compatibility with apps and software that support the HDHomeRun protocol, including Plex Live TV, Emby Live TV, Channels DVR, and more.

<Info>
  HDHomeRun endpoints are always publicly accessible and do not require authentication, ensuring compatibility with all HDHomeRun clients.
</Info>

## Device Discovery

Retrieve device information for HDHomeRun discovery.

```bash
GET /discover.json
```

### Response

<ResponseField name="FriendlyName" type="string">
  Human-readable device name
</ResponseField>

<ResponseField name="ModelNumber" type="string">
  HDHomeRun model identifier
</ResponseField>

<ResponseField name="FirmwareName" type="string">
  Firmware name
</ResponseField>

<ResponseField name="FirmwareVersion" type="string">
  ErsatzTV version
</ResponseField>

<ResponseField name="DeviceID" type="string">
  Unique device identifier
</ResponseField>

<ResponseField name="DeviceAuth" type="string">
  Device authentication token
</ResponseField>

<ResponseField name="BaseURL" type="string">
  Base URL for device API
</ResponseField>

<ResponseField name="LineupURL" type="string">
  URL to channel lineup
</ResponseField>

<ResponseField name="TunerCount" type="integer">
  Number of virtual tuners available
</ResponseField>

### Example Request

```bash
curl http://localhost:8409/discover.json
```

### Example Response

```json
{
  "FriendlyName": "ErsatzTV",
  "ModelNumber": "HDHR3-CC",
  "FirmwareName": "hdhomerun3_atsc",
  "FirmwareVersion": "0.8.9-beta",
  "DeviceID": "103E5C7B",
  "DeviceAuth": "ersatztv",
  "BaseURL": "http://localhost:8409",
  "LineupURL": "http://localhost:8409/lineup.json",
  "TunerCount": 3
}
```

### Implementation Details

From `ErsatzTV/Controllers/HdhrController.cs:22-28`:

```csharp
[HttpGet("discover.json")]
[ResponseCache(NoStore = true)]
public async Task<IActionResult> Discover()
{
    Guid uuid = await mediator.Send(new GetHDHRUUID());
    int tunerCount = await mediator.Send(new GetHDHRTunerCount());
    return new OkObjectResult(new Discover(Request.Scheme, Request.Host.ToString(), tunerCount, uuid));
}
```

The UUID is generated from a stable identifier and persisted across restarts. Tuner count is configurable in ErsatzTV settings.

## HDHomeRun Streaming

Channels are streamed via MPEG-TS format through dedicated HDHomeRun endpoints.

```bash
GET /iptv/hdhr/channel/{channelNumber}.ts
```

### Path Parameters

<ParamField path="channelNumber" type="string" required>
  Channel number to stream
</ParamField>

### Response

Returns a continuous MPEG-TS stream with `Content-Type: video/mp2t`.

### Example Request

```bash
curl http://localhost:8409/iptv/hdhr/channel/1.ts
```

### Implementation Details

From `IptvController.cs:70-81`:

```csharp
[HttpHead("iptv/hdhr/channel/{channelNumber}.ts")]
[HttpGet("iptv/hdhr/channel/{channelNumber}.ts")]
public Task<IActionResult> GetHDHRVideo(string channelNumber, [FromQuery] string mode = "ts")
{
    // don't redirect to the correct channel mode for HDHR clients; always use TS
    if (mode != "ts" && mode != "ts-legacy")
    {
        mode = "ts";
    }
    
    return GetTransportStreamVideo(channelNumber, mode);
}
```

HDHomeRun clients always receive MPEG-TS streams, regardless of the channel's configured streaming mode. This ensures compatibility with all HDHomeRun-compatible applications.

## Setting Up HDHomeRun Clients

### Plex Live TV

1. Navigate to **Settings** > **Live TV & DVR**
2. Click **Set Up Plex DVR**
3. Plex should auto-detect ErsatzTV as an HDHomeRun device
4. Select the device and complete setup

<Note>
  If auto-detection fails, use the IP address `http://localhost:8409` (replace with your server address)
</Note>

### Emby Live TV

1. Navigate to **Settings** > **Live TV**
2. Click **Add** under Tuner Devices
3. Select **HDHomeRun**
4. Enter ErsatzTV server address: `http://localhost:8409`
5. Complete the setup wizard

### Jellyfin Live TV

1. Navigate to **Dashboard** > **Live TV**
2. Click **Add** under TV Sources
3. Select **HDHomeRun**
4. Enter the tuner URL: `http://localhost:8409`
5. Complete configuration

### Channels DVR

1. Open Channels DVR settings
2. Navigate to **TV Sources**
3. Click **Add Source**
4. Select **HDHomeRun**
5. Enter the IP address of your ErsatzTV server
6. Channels should detect the lineup automatically

## Tuner Configuration

Configure the number of virtual tuners:

1. Navigate to **Settings** > **Playback** in ErsatzTV
2. Set **HDHomeRun Tuner Count**
3. Recommended: Set to the number of concurrent streams you want to support

<Info>
  Each "tuner" represents one concurrent stream. Set this based on your server's capacity and expected usage.
</Info>

## Discovery Protocols

ErsatzTV supports multiple discovery methods:

### HTTP Discovery

Direct HTTP access to `discover.json` and `device.xml`.

### SSDP/UPnP Discovery

ErsatzTV broadcasts SSDP announcements for automatic discovery by compatible clients.

<Note>
  For SSDP discovery to work, ensure your firewall allows UDP traffic on port 1900.
</Note>

## Troubleshooting

### Device Not Discovered

If clients can't find the ErsatzTV device:

1. Verify ErsatzTV is running and accessible
2. Check firewall rules (allow port 8409 and UDP 1900)
3. Manually add device using IP address
4. Ensure client and server are on the same network
5. Test with `curl http://YOUR_SERVER_IP:8409/discover.json`

### Channels Not Appearing

If channel lineup is empty:

1. Verify channels are enabled in ErsatzTV
2. Check that channels have valid schedules
3. Review channel streaming mode settings
4. Refresh lineup in client application

### Streaming Fails

If streams won't start:

1. Test stream URL directly with VLC
2. Check FFmpeg profile configuration
3. Verify transport stream mode is enabled
4. Review transcoding logs for errors
5. Ensure sufficient system resources

### Poor Performance

If streams lag or buffer:

1. Reduce concurrent stream count
2. Lower bitrate in FFmpeg profiles
3. Enable hardware acceleration
4. Check network bandwidth
5. Monitor server CPU and memory usage

## Network Configuration

### Port Requirements

- **8409/TCP** - HTTP API and streaming (default, configurable)
- **1900/UDP** - SSDP discovery

### Firewall Rules

Allow inbound traffic on:

```bash
# HTTP/streaming port
sudo ufw allow 8409/tcp

# SSDP discovery
sudo ufw allow 1900/udp
```

### Docker Configuration

For Docker deployments, expose ports:

```yaml
ports:
  - "8409:8409"
  - "1900:1900/udp"
```
