Integrate ErsatzTV with Emby to access your Emby libraries and include them in custom channels. ErsatzTV synchronizes metadata and media information directly from your Emby server.

## Overview

The Emby integration enables ErsatzTV to:

- Connect to Emby servers using API keys
- Synchronize movie and TV show libraries
- Access metadata, artwork, and playback information
- Support path replacements for different environments
- Handle multiple Emby servers simultaneously

## Authentication

### API Key Authentication

Emby uses API key-based authentication for third-party applications:

1. **Generate API Key**: Create an API key in the Emby dashboard
2. **Configure Connection**: Provide server address and API key to ErsatzTV
3. **Validate Server**: ErsatzTV verifies the connection and retrieves server details
4. **Synchronize Libraries**: Available libraries are discovered automatically

<CodeGroup>
```csharp SaveEmbySecretsHandler.cs
public Task<Either<BaseError, Unit>> Handle(
    SaveEmbySecrets request, 
    CancellationToken cancellationToken) =>
    Validate(request)
        .MapT(PerformSave)
        .Bind(v => v.ToEitherAsync());

private async Task<Validation<BaseError, Parameters>> Validate(
    SaveEmbySecrets request)
{
    Either<BaseError, EmbyServerInformation> maybeServerInformation = 
        await _embyApiClient.GetServerInformation(
            request.Secrets.Address, 
            request.Secrets.ApiKey
        );

    return maybeServerInformation.Match(
        info => Validation<BaseError, Parameters>.Success(
            new Parameters(request.Secrets, info)
        ),
        error => error
    );
}

private async Task<Unit> PerformSave(Parameters parameters)
{
    await _embySecretStore.SaveSecrets(parameters.Secrets);
    await _mediaSourceRepository.UpsertEmby(
        parameters.Secrets.Address,
        parameters.ServerInformation.ServerName,
        parameters.ServerInformation.OperatingSystem
    );
    _memoryCache.Remove(new GetEmbyConnectionParameters());
    await _channel.WriteAsync(new SynchronizeEmbyMediaSources());

    return Unit.Default;
}
```
</CodeGroup>

<Steps>

## Server Connection

### Server Information Retrieval

When establishing a connection, ErsatzTV retrieves server details:

<CodeGroup>
```csharp EmbyApiClient.cs
public async Task<Either<BaseError, EmbyServerInformation>> 
    GetServerInformation(
        string address,
        string apiKey)
{
    try
    {
        IEmbyApi service = RestService.For<IEmbyApi>(address);
        var cts = new CancellationTokenSource();
        cts.CancelAfter(TimeSpan.FromSeconds(5));
        
        return await service.GetSystemInformation(apiKey, cts.Token)
            .Map(response => new EmbyServerInformation(
                response.ServerName, 
                response.OperatingSystem
            ));
    }
    catch (Exception ex) when (ex is TaskCanceledException or 
                                    OperationCanceledException)
    {
        _logger.LogError(ex, "Timeout getting emby server name");
        return BaseError.New("Emby did not respond in time");
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "Error getting emby server name");
        return BaseError.New(ex.Message);
    }
}
```
</CodeGroup>

### Connection Features

- **Timeout Protection**: 5-second timeout for initial connection
- **Error Handling**: Comprehensive exception handling for network issues
- **Server Identification**: Captures server name and operating system
- **Validation**: Confirms API key has appropriate access

<CodeGroup>
```csharp EmbyApiClient.cs
public async Task<Either<BaseError, List<EmbyLibrary>>> GetLibraries(
    string address, 
    string apiKey)
{
    try
    {
        IEmbyApi service = RestService.For<IEmbyApi>(address);
        List<EmbyLibraryResponse> libraries = 
            await service.GetLibraries(apiKey);
        return libraries
            .Map(Project)
            .Somes()
            .ToList();
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "Error getting emby libraries");
        return BaseError.New(ex.Message);
    }
}
```
</CodeGroup>

```csharp EmbyApiClient.cs
public IAsyncEnumerable<Tuple<EmbyMovie, int>> GetMovieLibraryItems(
    string address,
    string apiKey,
    EmbyLibrary library)
    => GetPagedLibraryContents(
        address,
        library,
        library.ItemId,
```

### Pagination and Streaming

The Emby client uses efficient pagination:

- **Streaming API**: Uses `IAsyncEnumerable` for memory-efficient processing
- **Configurable Batches**: Adjustable page sizes for API requests
- **Progress Tracking**: Returns item counts for synchronization progress
- **Lazy Evaluation**: Processes items as they're retrieved

<CodeGroup>
```csharp EmbyApiClient.cs
public IAsyncEnumerable<Tuple<EmbyEpisode, int>> GetEpisodeLibraryItems(
    string address,
    string apiKey,
    EmbyLibrary library,
    string showId,
    string seasonId) => GetPagedLibraryContents(
    address,
    library,
    seasonId,
    (service, _, skip, pageSize) => service.GetEpisodeLibraryItems(
        apiKey,
        showId,
        seasonId,
        startIndex: skip,
        limit: pageSize),
    (maybeLibrary, item) => maybeLibrary
        .Map(lib => ProjectToEpisode(lib, item))
```

### Example Mappings

<CodeGroup>
```yaml Docker Setup
version: '3'
services:
  emby:
    volumes:
      - /srv/media:/media
  
  ersatztv:
    volumes:
      - /srv/media:/mnt/emby

# Configuration:
# Emby Path: /media/Movies
# Local Path: /mnt/emby/Movies
```

```yaml Network Share
# Emby sees NAS as: /mnt/nas/content
# ErsatzTV mounts it as: /media/nas

path_replacements:
  - emby_path: "/mnt/nas/content"
    local_path: "/media/nas"
```

```yaml Windows to Linux
# Emby on Windows: E:\\Videos
# ErsatzTV on Linux: /mnt/videos

path_replacements:
  - emby_path: "E:\\Videos"
    local_path: "/mnt/videos"
```
</CodeGroup>

### Path Replacement Service

The `IEmbyPathReplacementService` automatically applies path replacements:

- Translates Emby paths to local filesystem paths
- Handles multiple replacement rules
- Supports both prefix and full path replacements
- Caches replacement results for performance

## Disconnecting from Emby

To remove an Emby connection:

1. **Halt Synchronization**: Stop all active sync operations
2. **Remove Credentials**: Clear stored API key and server address
3. **Handle Content**: Choose to keep or remove synced media
4. **Update Search**: Refresh search index to remove entries

<Warning>
Disconnecting stops access to Emby libraries. Already-created channels can optionally preserve the content or be updated to remove Emby items.
</Warning>

## Troubleshooting

### Connection Problems

<AccordionGroup>
  <Accordion title="Cannot Connect to Emby Server">
    - Verify Emby server is running and accessible
    - Check server URL format (must include protocol: http:// or https://)
    - Confirm port number is correct (default: 8096)
    - Test connectivity: `curl http://emby-server:8096/System/Info`
    - Check firewall and network settings
  </Accordion>
  
  <Accordion title="Invalid API Key Error">
    - Verify API key was copied correctly without extra spaces
    - Check key hasn't been deleted in Emby dashboard
    - Ensure key has required permissions
    - Try generating a new API key
  </Accordion>
  
  <Accordion title="Connection Timeout">
    - Verify network latency between servers
    - Check if Emby server is overloaded
    - Increase timeout settings if on slow connection
    - Ensure no proxy or VPN interfering
  </Accordion>
</AccordionGroup>

### Synchronization Issues

<AccordionGroup>
  <Accordion title="Libraries Not Appearing">
    - Confirm API key has access to libraries
    - Check library type (only Movies and TV Shows sync)
    - Verify libraries contain content
    - Review Emby library permissions
    - Check ErsatzTV logs for errors
  </Accordion>
  
  <Accordion title="Missing Content">
    - Ensure Emby has scanned the library
    - Verify metadata is available in Emby
    - Check if content matches supported types
    - Manually trigger sync in ErsatzTV
  </Accordion>
  
  <Accordion title="Metadata Issues">
    - Verify Emby has scraped metadata
    - Check artwork URLs are accessible
    - Ensure metadata providers are configured
    - Re-identify content in Emby if needed
  </Accordion>
</AccordionGroup>

### Playback Problems

<AccordionGroup>
  <Accordion title="File Not Found">
    - Configure path replacements correctly
    - Verify ErsatzTV can access media files
    - Check filesystem permissions
    - Ensure mount points are active
    - Test file path with `ls` or `dir` command
  </Accordion>
  
  <Accordion title="Transcoding Failures">
    - Verify FFmpeg is installed and accessible
    - Check available disk space for transcoding
    - Review FFmpeg capabilities and codecs
    - Check ErsatzTV transcoding settings
  </Accordion>
</AccordionGroup>

### Performance Optimization

<AccordionGroup>
  <Accordion title="Slow Synchronization">
    - Reduce sync frequency for large libraries
    - Ensure both servers have adequate resources
    - Check network bandwidth and latency
    - Consider syncing libraries individually
    - Use scheduled sync during off-peak hours
  </Accordion>
  
  <Accordion title="High Resource Usage">
    - Limit concurrent sync operations
    - Reduce page size for API requests
    - Clear cache periodically
    - Monitor memory usage during sync
  </Accordion>
</AccordionGroup>
