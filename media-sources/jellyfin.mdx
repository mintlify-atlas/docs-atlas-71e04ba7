Integrate ErsatzTV with Jellyfin to access your Jellyfin libraries and include them in custom channels. ErsatzTV synchronizes metadata and media information directly from your Jellyfin server.

## Overview

The Jellyfin integration enables ErsatzTV to:

- Connect to Jellyfin servers via API
- Synchronize movie and TV show libraries
- Access metadata, artwork, and playback information
- Handle path replacements for containerized environments
- Support multiple Jellyfin servers

## Authentication

### API Key Authentication

Jellyfin uses API key-based authentication for external applications:

1. **Generate API Key**: Create an API key in Jellyfin dashboard
2. **Save Credentials**: Provide server address and API key to ErsatzTV
3. **Validate Connection**: ErsatzTV tests connection and retrieves server info
4. **Sync Libraries**: Available libraries are discovered and synchronized

<CodeGroup>
```csharp SaveJellyfinSecretsHandler.cs
public Task<Either<BaseError, Unit>> Handle(
    SaveJellyfinSecrets request, 
    CancellationToken cancellationToken) =>
    Validate(request)
        .MapT(PerformSave)
        .Bind(v => v.ToEitherAsync());

private async Task<Validation<BaseError, Parameters>> Validate(
    SaveJellyfinSecrets request)
{
    var connectionParameters = new JellyfinConnectionParameters(
        request.Secrets.Address, 
        request.Secrets.ApiKey, 
        0
    );

    Either<BaseError, JellyfinServerInformation> maybeServerInformation = 
        await _jellyfinApiClient.GetServerInformation(
            connectionParameters.Address, 
            connectionParameters.AuthorizationHeader
        );

    return maybeServerInformation.Match(
        info => Validation<BaseError, Parameters>.Success(
            new Parameters(request.Secrets, info)
        ),
        error => error
    );
}
```
</CodeGroup>

## Server Connection

### Server Information

When connecting, ErsatzTV retrieves server information:

<CodeGroup>
```csharp JellyfinApiClient.cs
public async Task<Either<BaseError, JellyfinServerInformation>> 
    GetServerInformation(
        string address,
        string authorizationHeader)
{
    try
    {
        IJellyfinApi service = ServiceForAddress(address);
        var cts = new CancellationTokenSource();
        cts.CancelAfter(TimeSpan.FromSeconds(5));
        
        return await service.GetSystemInformation(
                authorizationHeader, 
                cts.Token
            )
            .Map(response => new JellyfinServerInformation(
                response.ServerName, 
                response.OperatingSystem
            ));
    }
    catch (Exception ex) when (ex is TaskCanceledException or 
                                    OperationCanceledException)
    {
        _logger.LogError(ex, "Timeout getting jellyfin server name");
        return BaseError.New("Jellyfin did not respond in time");
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "Error getting jellyfin server name");
        return BaseError.New(ex.Message);
    }
}
```
</CodeGroup>

### Connection Validation

The connection process includes:

1. **Timeout Protection**: 5-second timeout for connection attempts
2. **Error Handling**: Graceful handling of connection failures
3. **Server Identity**: Captures server name and OS information
4. **Media Source Creation**: Creates or updates Jellyfin media source record

<CodeGroup>
```csharp JellyfinApiClient.cs
public async Task<Either<BaseError, List<JellyfinLibrary>>> GetLibraries(
    string address, 
    string authorizationHeader)
{
    try
    {
        IJellyfinApi service = ServiceForAddress(address);
        List<JellyfinLibraryResponse> libraries = 
            await service.GetLibraries(authorizationHeader);
        return libraries
            .Map(Project)
            .Somes()
            .ToList();
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "Error getting jellyfin libraries");
        return BaseError.New(ex.Message);
    }
}
```
</CodeGroup>

### Supported Library Types

ErsatzTV synchronizes:

- **Movie Libraries**: All movie-type content
- **TV Show Libraries**: Series with seasons and episodes
- **Collections**: Jellyfin collections (playlists/smart collections)

### Content Synchronization

For each library type, ErsatzTV retrieves:

#### Movies
- Title, year, and sort title
- Genres, studios, and tags
- Ratings and content ratings
- Poster and backdrop artwork
- Media file information and codecs

#### TV Shows
- Show metadata and overview
- Season information
- Episode details with air dates
- Actor, director, and writer information
- Episode thumbnails and artwork

### Pagination Strategy

ErsatzTV uses efficient pagination for large libraries:

<CodeGroup>
```csharp JellyfinApiClient.cs
public IAsyncEnumerable<Tuple<JellyfinMovie, int>> GetMovieLibraryItems(
    string address,
    string authorizationHeader,
    JellyfinLibrary library) =>
    GetPagedLibraryItems(
        "JF Movies",
        address,
        library,
        library.MediaSourceId,
        library.ItemId,
        (service, itemId, skip, pageSize) => service.GetMovieLibraryItems(
            authorizationHeader,
            itemId,
            startIndex: skip,
            limit: pageSize),
        (maybeLibrary, item) => maybeLibrary
            .Map(lib => ProjectToMovie(lib, item))
            .Flatten());
```
</CodeGroup>

- **Streaming Results**: Uses `IAsyncEnumerable` for memory efficiency
- **Configurable Page Size**: Adjustable batch sizes for API requests
- **Progress Tracking**: Returns item counts for progress reporting

### Example Configurations

<CodeGroup>
```yaml Docker Compose Example
services:
  jellyfin:
    volumes:
      - /mnt/media:/media
  
  ersatztv:
    volumes:
      - /mnt/media:/data/media

# Path Replacement:
# Jellyfin Path: /media/movies
# Local Path: /data/media/movies
```

```yaml Network Share Example
# Jellyfin sees: /mnt/nas/Movies
# ErsatzTV sees: /media/movies

path_replacements:
  - jellyfin_path: "/mnt/nas/Movies"
    local_path: "/media/movies"
```

```yaml Windows to Linux Example
# Jellyfin Path: C:\\Media\\Shows
# Local Path: /mnt/media/shows

path_replacements:
  - jellyfin_path: "C:\\Media\\Shows"
    local_path: "/mnt/media/shows"
```
</CodeGroup>

Customize how each Jellyfin library is managed:

## Disconnecting from Jellyfin

To remove a Jellyfin connection:

1. **Stop Synchronization**: Halts all active sync operations
2. **Clear Secrets**: Removes stored API key and address
3. **Preserve Media**: Option to keep or remove synced content
4. **Clean Search Index**: Updates search index as needed

<Warning>
Disconnecting removes access to Jellyfin libraries but can optionally preserve already-synced content in channels.
</Warning>

## Troubleshooting

### Connection Issues

<AccordionGroup>
  <Accordion title="Cannot Connect to Server">
    - Verify Jellyfin server is running
    - Check server address format (include http:// or https://)
    - Ensure port is correct (default: 8096)
    - Test network connectivity from ErsatzTV host
  </Accordion>
  
  <Accordion title="Invalid API Key">
    - Verify API key was copied correctly
    - Check API key hasn't been revoked in Jellyfin
    - Ensure API key has necessary permissions
    - Generate a new API key if needed
  </Accordion>
  
  <Accordion title="Timeout Errors">
    - Increase timeout settings if on slow network
    - Check for firewall blocking connections
    - Verify Jellyfin server isn't overloaded
  </Accordion>
</AccordionGroup>

### Synchronization Issues

<AccordionGroup>
  <Accordion title="Libraries Not Syncing">
    - Verify API key has access to libraries
    - Check Jellyfin library permissions
    - Review ErsatzTV logs for errors
    - Manually trigger library sync
  </Accordion>
  
  <Accordion title="Missing Metadata">
    - Ensure Jellyfin has scraped metadata
    - Verify artwork is accessible
    - Check metadata provider configuration
    - Re-scan library in Jellyfin first
  </Accordion>
  
  <Accordion title="Path Issues">
    - Configure path replacements correctly
    - Verify file paths are accessible to ErsatzTV
    - Check mount points in Docker environments
    - Test file access permissions
  </Accordion>
</AccordionGroup>

### Performance Optimization

<AccordionGroup>
  <Accordion title="Slow Synchronization">
    - Reduce library scan frequency
    - Sync only changed items when possible
    - Check network bandwidth between servers
    - Ensure Jellyfin server has adequate resources
  </Accordion>
  
  <Accordion title="High Memory Usage">
    - Pagination helps but very large libraries may need tuning
    - Reduce concurrent sync operations
    - Clear cached data periodically
  </Accordion>
</AccordionGroup>
